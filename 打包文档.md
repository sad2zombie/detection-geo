## 1. 系统架构

```
Electron 主进程（main.js）
    │
    ├── 启动 Python 后端子进程  detection-backend-exe.exe
    │       ├── 无控制台窗口（console=False）
    │       ├── 监听端口 8000（http://127.0.0.1:8000）
    │       └── 数据目录：%APPDATA%/detection/data/
    │
    └── 加载前端页面（http://127.0.0.1:8000）
            ├── web/templates/     HTML/Jinja2 模板
            └── web/static/        CSS/JS/图片

后端技术栈：FastAPI + Uvicorn + CloakBrowser（stealth Chromium）
前端技术栈：Electron 28 + preload.js + BrowserWindow
```

**关键路径**：
- `main.py` → 入口，由 `backend-exe.spec` 打包为 exe
- `web/server.py` → FastAPI 应用，注册路由、渲染模板
- `config.py` → 全局配置（CloakBrowser 路径、数据目录、平台 Cookie 路径）
- `core/browser_manager.py` → CloakBrowser 浏览器管理

## 3. 前置条件

### 3.1 Python 环境

- Python 3.10+，建议虚拟环境
- 安装核心依赖（与开发环境一致）：

```powershell
pip install fastapi uvicorn jinja2 cloakbrowser
```

### 3.2 CloakBrowser Chromium 二进制

**重要**：打包前必须确保本地已下载 CloakBrowser 的 Chromium，否则打包进去的二进制不存在，运行时需要联网。

```powershell
# 下载 CloakBrowser Chromium（约 400MB，跳过则运行时联网下载）
python -c "from cloakbrowser.download import ensure_binary; ensure_binary()"

# 确认下载成功
dir %USERPROFILE%\.cloakbrowser
```

下载后，`%USERPROFILE%\.cloakbrowser\chromium-<版本>\chrome.exe` 会作为数据文件打包进 exe。

### 3.3 Node.js 环境

- Node.js 16+，npm 已安装
- 安装 Electron 及 electron-builder：

```powershell
npm install
```

### 3.4 PyInstaller

```powershell
pip install pyinstaller
```

## 4. 打包步骤

### 4.1 步骤一：打包 Python 后端（PyInstaller）

```powershell
cd C:\Users\Administrator\PycharmProjects\detection

pyinstaller backend-exe.spec
```

**预期产出**：`dist/detection-backend-exe.exe`

- 无控制台窗口（`console=False`）
- 大小约 500–600 MB（含 CloakBrowser 源码 + Chromium 二进制）
- 打包时 `runtime_hook.py` 由 spec 自动生成

**注意**：每次重新打包前，建议清理旧 build 目录以避免缓存问题：

```powershell
Remove-Item -Recurse -Force build, dist -ErrorAction SilentlyContinue
pyinstaller backend-exe.spec
```

### 4.2 步骤二：打包 Electron 安装程序（electron-builder）

```powershell
npm run build
```

**预期产出**：`release/品牌检测 Setup 1.0.0.exe`

electron-builder 内部会自动：
1. 把 `main.js`、`preload.js`、`web/**/*`、`package.json` 打包进 `app.asar`
2. 从 `dist/detection-backend-exe.exe` 复制到 `resources/detection-backend-exe.exe`
3. 生成 NSIS 安装程序

安装程序支持：
- 自定义安装路径（`allowToChangeInstallationDirectory: true`）
- 创建桌面快捷方式
- 创建开始菜单快捷方式

### 4.3 快速打包脚本

在项目根目录新建 `build.bat`，一键完成两步：

```bat
@echo off
chcp 65001 >NUL
echo [1/2] PyInstaller...
Remove-Item -Recurse -Force build, dist -ErrorAction SilentlyContinue
pyinstaller backend-exe.spec
if %errorlevel% neq 0 (
    echo [ERROR] PyInstaller failed.
    pause
    exit /b 1
)
echo [2/2] electron-builder...
npm run build
if %errorlevel% neq 0 (
    echo [ERROR] electron-builder failed.
    pause
    exit /b 1
)
echo.
echo [OK] 打包完成！安装包位于 release/ 目录。
pause
```

## 6. 重要配置说明

### 6.1 backend-exe.spec 关键配置

| 配置项 | 值 | 说明 |
|---|---|---|
| `name` | `"detection-backend-exe"` | exe 文件名（注意与 media-distribute-record 区分） |
| `console` | `False` | 无控制台窗口，后台运行 |
| `runtime_hooks` | `runtime_hook.py` | 打包时自动生成，设置 CloakBrowser 路径 |
| `datas` | `config.py`, `web/templates`, `web/static`, `core/`, `platforms/`, `utils/`, `ai/`, `cloakbrowser/` | 业务代码及前端资源 |
| `hiddenimports` | `uvicorn`, `fastapi`, `cloakbrowser`, `pydantic`, 各业务模块 | PyInstaller 无法静态分析到的隐式导入 |

**runtime_hook.py 的职责**（打包时由 spec 动态生成）：
1. 把 `_MEIPASS/cloakbrowser` 加到 `sys.path`，使 `import cloakbrowser` 在 frozen 环境可用
2. 设 `CLOAKBROWSER_BINARY_PATH` 指向打包的 `chrome.exe`，跳过在线下载
3. 设 `CLOAKBROWSER_CACHE_DIR` 到 `%LOCALAPPDATA%\cloakbrowser-cache`，供可写操作使用

### 6.2 main.js 关键配置

| 行 | 配置项 | 值 |
|---|---|---|
| 43 | 子进程 exe 路径（打包模式） | `process.resourcesPath/detection-backend-exe.exe` |
| 58-63 | 数据目录（打包模式） | `%APPDATA%/detection/data/` |
| 76 | 子进程启动参数 | `detached: false, windowsHide: true` |
| 173 | 后端健康检查地址 | `http://127.0.0.1:8000` |
| 206, 266 | 关闭时杀进程 | `taskkill /F /IM detection-backend-exe.exe` |

### 6.3 package.json electron-builder 配置

| 字段 | 值 | 说明 |
|---|---|---|
| `productName` | `"品牌检测"` | 安装后的显示名 |
| `asar` | `false` | 不使用 asar 打包（直接解压所有文件） |
| `extraResources[0].from` | `"dist/detection-backend-exe.exe"` | 从这里复制 |
| `extraResources[0].to` | `"detection-backend-exe.exe"` | 到 `resources/detection-backend-exe.exe` |
| `nsis.oneClick` | `false` | 不一键安装，允许选路径 |
| `nsis.allowToChangeInstallationDirectory` | `true` | 允许修改安装目录 |

### 6.4 config.py 关键路径

| 变量 | 开发环境 | 打包环境 |
|---|---|---|
| `CLOAKBROWSER_DIR` | `%CLOAKBROWSER_DIR%` → 项目根目录 `cloakbrowser/` | `_MEIPASS/cloakbrowser/`（runtime_hook 已注入） |
| `DATA_DIR` | `项目根目录/data/` | `%APPDATA%/detection/data/`（Electron 启动时注入 `DETECTION_DATA_DIR`） |
| 后端端口 | `8000` | `8000`（`main.py`/`web/server.py` 写死） |

### 6.5 为什么 exe 这么大（500MB+）

```
detection-backend-exe.exe  ≈  550 MB
├── Python 运行时 + 标准库（约 30 MB）
├── 业务代码（core/platforms/utils/ai + web/templates/static）（约 10 MB）
├── CloakBrowser 源码（pip 包，约 150 MB）
└── Chromium 二进制（chrome.exe + 资源，约 350 MB）
```

## 7. 常见问题

### Q1：PyInstaller 打包后启动报错 `ModuleNotFoundError: No module named 'xxx'`

**原因**：某些动态导入的模块未被 PyInstaller 检测到。

**解决**：在 `backend-exe.spec` 的 `hiddenimports` 列表中添加该模块名，然后重新打包。

```python
hiddenimports = [
    # ... 已有模块 ...
    "缺失的模块名",
]
```

### Q2：CloakBrowser 运行时提示需要下载 chromium

**原因**：`%USERPROFILE%\.cloakbrowser\` 下没有已下载的 Chromium，打包时也没有打包进去。

**解决**：
```powershell
python -c "from cloakbrowser.download import ensure_binary; ensure_binary()"
```
确认 `chrome.exe` 存在后再重新打包。

### Q3：Electron 安装后启动报错 `spawn detection-backend-exe.exe error`

**可能原因**：
1. 杀毒软件拦截（Avast/360/Windows Defender）→ 将 `品牌检测` 加入白名单
2. 安装路径有特殊字符 → 重装到不含中文/空格的路径

### Q4：想改后端端口（比如改成 8001）

**原因**：本项目（detection）与 media-distribute-record 都用 8000 端口，避免冲突。

**操作**：改以下 3 处：
- `web/server.py` 第 140 行：`port=8000` → `port=8001`
- `main.py` 第 47 行：`port=8000` → `port=8001`
- `main.js` 第 173 行：`http://127.0.0.1:8000` → `http://127.0.0.1:8001`

然后重新打包两步（PyInstaller → electron-builder）。

### Q5：打包后如何验证 exe 能正常运行

在打包完成后、安装前，先单独运行 exe 确认：

```powershell
# 启动后端（无窗口，会在后台运行）
.\dist\detection-backend-exe.exe

# 等待 5 秒后检查端口是否监听
Start-Sleep -Seconds 5
netstat -ano | Select-String ":8000"

# 如果 LISTENING 状态出现，说明后端启动成功，按 Ctrl+C 停止

# 也可以用浏览器直接访问验证
# 浏览器打开 http://127.0.0.1:8000
```

### Q6：taskkill 杀错了进程（误杀了 media-distribute-record 的后端）

**原因**：exe 重名前，两个项目的进程名都是 `backend-exe.exe`，`taskkill /IM backend-exe.exe` 会同时杀掉两个。

**解决**：已完成 —— 本项目已改名为 `detection-backend-exe.exe`，`taskkill /IM detection-backend-exe.exe` 只杀本项目。

### Q7：升级版本号

`package.json` 第 3 行：`"version": "1.0.0"` 改为 `"1.1.0"`，重新 `npm run build`，生成的安装包文件名自动包含新版本号。
